# 學生報名

**路徑**: `/admin/enrollment`
**角色**: Admin
**分組**: 報名管理

## 核心目的

管理員直接將學生加入開課班（跳過申請流程）。

## MVP 功能

### 直接報名流程

1. 選擇學生（可搜尋）
2. 選擇開課班
3. 設定繳費週期（月繳/期繳）
4. 設定生效起迄日期
5. 選擇是否建立繳費單
   - 是：建立繳費單，狀態為「待付款」
   - 否：直接建立報名，不產生繳費單
6. 確認

### 報名列表管理

- 篩選：學生、開課班、狀態、日期範圍
- 每筆顯示：學生、開課班、生效期間、繳費週期、狀態
- 操作：編輯、終止報名

### 報名狀態

- `pending_payment`：待付款
- `active`：正式生效
- `suspended`：停權
- `withdrawal`：退選
- `void`：失效

### 狀態變更

- 變更為非活躍狀態時，必須填寫原因

## 資料依賴

| 操作 | 資料表 |
|------|--------|
| 讀取 | `enrollments`, `students`, `classes`, `fee_templates` |
| 寫入 | `enrollments`, `invoices`（可選） |

## 實作註記

- 學生沒有報名開課班就不會出現在任何課堂的學生名單中
- 管理員直接報名可選擇是否產生繳費單，方便處理已現場繳費或特殊免費情況
